# Routing Rules

## Route work to Lead when
- the request changes scope, priorities, or MVP boundaries
- the request introduces trade-offs across multiple workload choices
- a decision is required on cost, complexity, or demo value

## Route work to Solution Architect when
- the request concerns medallion design, item selection, Fabric workload choice, or environment topology
- the request concerns canonical entity design, shared schemas, or KPI-serving architecture

## Route work to Batch Ingestion Engineer when
- the request concerns API pulls, notebook ingestion, pipeline orchestration, scheduling, retry logic, or parameterization

## Route work to Streaming Engineer when
- the request concerns Eventstreams, Eventhouse/KQL, stream transformations, or real-time telemetry patterns

## Route work to Transformation Engineer when
- the request concerns Dataflow Gen2, low-code shaping, standardization, and Silver-layer cleanup

## Route work to SQL / Warehouse Engineer when
- the request concerns Materialized Lake Views, Warehouse model design, fact/dimension patterns, SQL performance, or KPI mart views

## Route work to DemoOps / QA Engineer when
- the request concerns runbooks, smoke tests, CI/CD, Git integration, environment variables, observability, and demo reliability

## Route work to Scribe when
- the request concerns docs, decision logs, architecture notes, talk track support, storytelling, or task tracking
